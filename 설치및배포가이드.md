## 🖥️ 로컬 환경에서 실행하기

### **Step 1: Python 설치 확인**
Python 3.8 이상이 설치되어 있는지 확인합니다.

```bash
python --version
# 또는
python3 --version
```

버전이 3.8 이상이면 OK! ✅

### **Step 2: 프로젝트 폴더 구조**
다음과 같이 폴더를 만드세요:

```
attendance_dashboard/
├── app.py                    (메인 프로그램 파일)
├── requirements.txt          (라이브러리 설정 파일)
├── .streamlit/               (Streamlit 설정 폴더 - 자동 생성됨)
└── attendance_data/          (데이터 저장 폴더 - 자동 생성됨)
    └── attendance.csv        (데이터 파일 - 자동 생성됨)
```

### **Step 3: 라이브러리 설치**
터미널/명령프롬프트를 열고 프로젝트 폴더로 이동한 후:

```bash
# Windows
cd C:\Users\YourName\Desktop\attendance_dashboard
pip install -r requirements.txt

# Mac / Linux
cd ~/Desktop/attendance_dashboard
pip3 install -r requirements.txt
```

**설치가 완료되면:**
```
Successfully installed streamlit pandas numpy openpyxl
```

### **Step 4: 앱 실행**

```bash
# Windows / Mac / Linux 공통
streamlit run app.py
```

**성공 메시지:**
```
  Local URL: http://localhost:8501
  Network URL: http://192.168.x.x:8501
```

브라우저가 자동으로 열리고 `http://localhost:8501`이 나타나면 성공! 🎉

### **Step 5: 로컬에서 테스트**

#### 부서 관리자 모드 테스트:
1. 좌측 사이드바 → "📋 부서 관리자 모드" 선택
2. 부서 드롭다운에서 "회계부" 선택
3. 직원 목록이 나타남 ✅
4. "실제출근" 시간 입력 (예: 08:30)
5. "근무상태" 선택 (예: 출근완료)
6. 💾 저장 버튼 클릭
7. "✅ 데이터가 저장되었습니다!" 메시지 확인 ✅

#### 총괄 관리자 모드 테스트:
1. 좌측 사이드바 → "📊 총괄 관리자 모드" 선택
2. 상단에 "📊 출근율", "✅ 출근완료" 등의 카드 확인 ✅
3. "🏢 부서별 상세 현황" 표 확인 ✅
4. 👥 전체 직원 리스트 스크롤해서 확인
5. "📊 현황 엑셀로 다운로드" 버튼 클릭 → xlsx 파일 다운로드 확인 ✅

### **Step 6: 앱 종료**
터미널에서 `Ctrl+C` 누르면 종료됩니다.

## 🚀 Streamlit Cloud 배포하기

### **Step 1: GitHub 회원가입 (무료)**

1. [https://github.com/](https://github.com/) 접속
2. "Sign up" 클릭
3. 이메일, 비밀번호 입력하고 회원가입
4. 이메일 인증 완료

### **Step 2: GitHub 저장소(Repository) 생성**

1. GitHub 로그인 후 우측 상단 + 아이콘 → "New repository" 클릭
2. Repository 이름: `attendance-dashboard`
3. Description: `보물섬수산 출근현황 관리 대시보드`
4. "Public" 선택 (배포할 수 있도록)
5. "Create repository" 클릭

### **Step 3: Git 설치 (Windows/Mac/Linux)**

#### Windows:
- [https://git-scm.com/download/win](https://git-scm.com/download/win) 다운로드 → 설치
- 기본 옵션으로 진행

#### Mac:
```bash
brew install git
```

#### Linux:
```bash
sudo apt-get install git
```

**설치 확인:**
```bash
git --version
# git version 2.x.x 이상
```

### **Step 4: Git 초기 설정**

```bash
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
```

### **Step 5: 프로젝트 폴더를 Git 저장소로 만들기**

```bash
cd attendance_dashboard

# Git 초기화
git init

# 모든 파일 추가
git add .

# 첫 번째 커밋
git commit -m "Initial commit: Attendance Dashboard"
```

### **Step 6: GitHub에 업로드**

GitHub 저장소 페이지에서:
1. "…or push an existing repository from the command line" 섹션의 코드 복사
2. 로컬 터미널에 붙여넣기 및 실행

예시:
```bash
git branch -M main
git remote add origin https://github.com/YourUsername/attendance-dashboard.git
git push -u origin main
```

**완료 메시지:**
```
Enumerating objects: 5, done.
Counting objects: 100% (5/5), done.
...
Branch 'main' set up to track remote branch 'main' from 'origin'.
```

### **Step 7: Streamlit Cloud 계정 생성**

1. [https://streamlit.io/cloud](https://streamlit.io/cloud) 접속
2. "Sign up" → GitHub로 로그인 (GitHub 계정과 연동)
3. Streamlit의 GitHub 접근 권한 허용

### **Step 8: 앱 배포**

1. Streamlit Cloud 대시보드에서 "New app" 클릭
2. 다음 정보 입력:
   - **Repository**: `YourUsername/attendance-dashboard`
   - **Branch**: `main`
   - **Main file path**: `app.py`
3. "Deploy!" 클릭

### **Step 9: 배포 진행 상황 모니터링**

배포 중:
```
📦 Installing dependencies...
🚀 Building app...
✅ App is now live!
```

배포 완료 후:
```
Your app is live at:
https://attendance-dashboard-xxxxx.streamlit.app
```

### **Step 10: 배포된 앱 공유**

#### 내부 공유 (직원용):
```
📌 출근현황 관리 대시보드 링크:
https://attendance-dashboard-xxxxx.streamlit.app
```

이 링크를 직원들에게 공유! 
- 설치 필요 없음
- 어디서든 접속 가능
- 자동 저장 (로컬 CSV에 저장됨)

모바일 브라우저에서 위 링크 접속하면 자동으로 반응형 레이아웃 적용! ✅

## 🔄 앱 업데이트하기

로컬 코드를 수정한 후 배포된 앱에 반영하려면:

```bash
# 1. 로컬에서 코드 수정
# app.py 또는 다른 파일 수정...

# 2. 변경사항 커밋
git add .
git commit -m "Update: 수정 내용 설명"

# 3. GitHub에 푸시
git push

# 4. Streamlit Cloud가 자동으로 감지하고 재배포
# (대시보드에서 "Redeploy" 버튼이 나타나면 클릭 가능)
```

Streamlit Cloud는 GitHub 저장소의 변경을 **자동 감지**하므로 수동 재배포도 가능! ✅

## 🛠️ 트러블슈팅

### **❌ 문제: "command not found: streamlit"**

**해결:**
```bash
# pip 경로 확인
python -m pip install -r requirements.txt

# 또는
pip3 install streamlit
```

### **❌ 문제: "ModuleNotFoundError: No module named 'streamlit'"**

**해결:**
```bash
# 가상환경 사용 권장 (선택사항)
python -m venv venv

# Windows
venv\Scripts\activate

# Mac/Linux
source venv/bin/activate

# 라이브러리 재설치
pip install -r requirements.txt
```

### **❌ 문제: 포트 8501이 이미 사용 중**

**해결:**
```bash
# 다른 포트로 실행
streamlit run app.py --server.port 8502
```

### **❌ 문제: GitHub에 푸시할 때 인증 오류**

**해결 (Personal Access Token 사용):**
1. GitHub 우측 상단 프로필 → Settings → Developer settings → Personal access tokens
2. "Generate new token" → repo 권한만 선택 → "Generate token"
3. 토큰 복사 (다시는 안 보임!)
4. 터미널에서:
   ```bash
   git push
   # Username: your-username
   # Password: (복사한 토큰 붙여넣기)
   ```

### **❌ 문제: Streamlit Cloud 배포 중 오류**

**해결:**
1. Streamlit Cloud 대시보드에서 "Logs" 탭 확인
2. 에러 메시지 읽고 requirements.txt 버전 확인
3. 필요하면:
   ```bash
   pip freeze > requirements.txt
   git add requirements.txt
   git commit -m "Fix: requirements.txt"
   git push
   ```

### **❌ 문제: 데이터가 저장되지 않음**

**확인 사항:**
1. `attendance_data/` 폴더가 자동으로 생성되었는가?
2. 폴더 권한 확인 (읽기/쓰기 가능한가?)
3. 디스크 공간 충분한가?

**해결 (권한 리셋):**
```bash
# Mac/Linux
chmod 755 attendance_data
chmod 644 attendance_data/attendance.csv

# Windows는 보통 자동 처리됨
```
